# Milestone v7.0.0: Per-Replica Processor Liveness & Self-Watchdog

**Status:** ✅ CLOSED 2026-06-14 (audit-override — see Known Gaps)
**Phases:** 59–62 (+ 62.1 inserted)
**Total Plans:** 12 (Phases 59–61 + 62.1) authored & hermetically green; Phase 62 (3 plans) authored, live close gate not run

## Overview

Make processor liveness multi-replica-accurate and self-healing. Replace the single last-write-wins L2 liveness key `skp:{processorId}` with **per-instance keys** `skp:proc:{processorId}:{instanceId}` under a per-processor **instance-index SET** `skp:proc:{processorId}` (TTL = source of truth; index = discovery hint, lazily `SREM`'d when stale). The liveness value gains a **two-state `status`** (`healthy`/`unhealthy`) + a per-schema **`summary`** (`{inputSchema, outputSchema, configSchema ∈ SUCCESS|FAIL}`; any FAIL ⇒ unhealthy; `configSchema` reuses v6.0.0 Gate A's outcome) and is written by **BOTH** the startup loop (from its first iteration — so L2 reflects a *restarting* replica as `unhealthy`, never absent) and the heartbeat loop (timestamp refresh only; health frozen `healthy` once heartbeat starts). An **in-memory L1 liveness record** is added (updated by both loops). The WebAPI orchestration-start gate becomes a **≥1-healthy-and-fresh** check across replicas. A **self-watchdog liveness probe** reads the L1 record's staleness so a silently-crashed loop ⇒ stale L1 ⇒ probe fails ⇒ (future) K8s pod restart. `inputDefinition`/`outputDefinition` are **dropped from L2**. **Breaking change** to the processor liveness contract; builds directly on v6.0.0 Gate A.

**Build order (locked, dependency-driven):** 59 (per-instance keyspace + two-state value) → 60 (dual-loop writer + in-memory L1 + unhealthy-is-written) → 61 (WebAPI ≥1-healthy gate + self-watchdog probe) → 62 (live proof + close gate).

## Milestone Summary

**Decimal Phases:**
- Phase 62.1: Decouple liveness refresh from IsHealthy (inserted after Phase 62 to fix gap G-62-01 — refresh was IsHealthy-gated, so an alive-but-unhealthy replica's L2 gate key expired and its L1 watchdog went stale → false restart).

**Key Decisions:**
- Per-instance L2 keys + instance-index SET (TTL = source of truth; index = lazily-pruned discovery hint) — replaces single last-write-wins key.
- Two-state health + per-schema summary; startup loop writes `unhealthy` (restarting replica visible, never absent); frozen-healthy after heartbeat starts.
- WebAPI gate = ≥1-healthy-and-fresh across replicas (presence no longer implies live).
- Liveness self-watchdog reads in-memory L1 staleness (probe semantics built; K8s wiring deferred).
- Liveness refresh decoupled from IsHealthy (G-62-01 fix).
