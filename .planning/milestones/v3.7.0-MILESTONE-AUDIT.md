## Milestone Definition of Done

> Make the autonomous execution loop (cron fire -> dispatch -> process -> result -> fan-out) self-heal
> through transient L2 (Redis) outages without operator intervention. A multi-replica Keeper console
> reacts to Fault<EntryStepDispatch> / Fault<ExecutionResult> events, probes L2 health on a bounded
> loop, pauses the affected workflow's cron, re-injects recovered work to its origin (riding the
> receiver's flag[H] idempotency), resumes when no recoveries remain, and parks the genuinely
> unrecoverable in keeper-dlq for operator triage.

**Verdict: ACHIEVED + HARDENED.** The recover-both-paths and give-up flows are proven live (Phase-39
close gate, 3x consecutive GREEN at 500 RealStack facts each, triple-SHA net-zero held). Phases 40-41
added the recovery attempt-cap, deterministic drain, and pause/resume diagnostics; Phase 42 reconciled
the docs. All cross-phase wires connect at source level (integration CLEAN).

0 orphaned, 0 broken flows):

   published before probe (37) -> bounded L2ProbeRecovery loop (36) -> re-inject-by-type via
   StepDispatcher with flag[H] idempotency (33/36) -> Resume on Recovered (37) -> park to keeper-dlq
   on give-up with attempt cap (36/40). WIRED.

   same KeeperRecoveryHandler that owns the probe-loop delegation; recover_cap park converges with
   probe_exhausted park on keeper-dlq. WIRED.

   transport DLQ (7d TTL) declared once in BaseConsole.Core; Keeper recovers off the Fault<T> pub/sub
   stream, not the _error queue. WIRED.
4. **Metrics uniformity** - combined service_name + non-empty service_instance_id on all three
   services; Keeper meters carry bounded labels with no workflowId/correlationId cardinality leak;
   processor DB-sourced name via MeterProviderHolder swap. An MLBL-04 regression (versioned name
