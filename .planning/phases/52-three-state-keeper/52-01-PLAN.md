```
DELETE(entryId):  delete L2[entryId]; if NOT exist -> drop
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| broker -> keeper recovery consumer | Typed MassTransit envelopes (KeeperReinject/Inject/Delete) from trusted internal services; no external/user input. |
| keeper -> L2 (Redis) | L2 read/write/delete of execution-data blobs (opaque JSON). |
| keeper -> broker (Send) | Reconstructed EntryStepDispatch / StepCompleted sent forward. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-52-01 | Information disclosure | D-07 drop log in ReinjectConsumer | mitigate | Structured `{EntryId}` hole ONLY; NEVER log m.Payload/m.Data contents (V7, PauseAllConsumer convention). Acceptance grep ensures no `$"..."` interpolation of payload. |
| T-52-02 | Tampering / data-loss correctness | REINJECT silent-drop path (D-06) | mitigate | Drop is gated on STRLEN==0 with NO Redis exception; a Redis EXCEPTION must still propagate through Guard to the exhaustion policy (NOT swallowed). Behavior fact asserts the exception path is not a drop. A18 lists data-gone as an accepted silent loss (data genuinely gone -> replay impossible -> nothing downstream lost). |
| T-52-03 | Tampering / out-of-order effect | INJECT op ordering (Pitfall 5) | mitigate | Strict write->send->delete order; delete is the tail AFTER the confirmed send. `Received.InOrder` fact locks the order so a completed result is never lost by deleting the source before the send lands. |
| T-52-04 | Denial of service | counter leakage across hermetic test process | accept | KeeperMetrics uses IMeterFactory (not static Meter), preventing cross-test leakage; no external DoS surface (internal consumer). |
</threat_model>
