The method signature is `private async Task RunRecoveryAsync(EntryStepDispatch d, Guid messageId, IDatabase db, int limit, CancellationToken ct)` (match whatever plan 02 declared; if plan 02 passed `executionDataTtl`, drop it — recovery does not write data keys).

    ```csharp
    // RECOV-01: HGETALL the slot array. Exhaust → REINJECT; end (no source delete).
    var read = await RetryLoop.ExecuteAsync(
        () => db.HashGetAllAsync(L2ProjectionKeys.MessageIndex(messageId)), limit, ct);
    if (!read.Succeeded)
    {
        await SendKeeper(BuildReinject(d), limit, ct);
        return;
    }

    // Build the temp list per slot. Claude's-Discretion representation: a local list of
    // (RedisValue slotField, Guid entryId, Outcome, bool infra).
    var temp = new List<(RedisValue Slot, Guid EntryId, bool Completed, bool Infra)>();
    foreach (var entry in read.Value!)
    {
        if (!Guid.TryParse(entry.Value.ToString(), out var entryId) || entryId == Guid.Empty)
            continue;   // already-retired (guid.empty) or unparsable slot — skip (A18: retired slots are inert)

        var exist = await RetryLoop.ExecuteAsync(
            () => db.KeyExistsAsync(L2ProjectionKeys.ExecutionData(entryId)), limit, ct);

        if (!exist.Succeeded)
            temp.Add((entry.Name, entryId, Completed: false, Infra: true));    // L2-fail → infra_entryId (leave slot)
        else if (exist.Value)
            temp.Add((entry.Name, entryId, Completed: true,  Infra: false));   // exists → completed
        // clean not-exist → failed/drop: NOT added (no send, no retire)
    }

    // Dispatch + send-before-retire (SLOT-03).
    foreach (var t in temp)
    {
        if (!t.Completed) continue;   // infra_entryId temp items: leave slot intact, no send (handled in the tail)

        // D-03/Pitfall 4: recovery completed mints a FRESH exec (slot holds only entryId, no exec persisted).
        await SendResult(BuildCompleted(d, NewId.NextGuid(), t.EntryId), limit, ct);   // SEND FIRST

        // retire AFTER a confirmed send (SendResult throws on send-exhaust, so reaching here == sent).
        var retire = await RetryLoop.ExecuteAsync(
            () => db.HashSetAsync(L2ProjectionKeys.MessageIndex(messageId), t.Slot, Guid.Empty.ToString()), limit, ct);
        if (retire.Succeeded)
        {
            await RetryLoop.ExecuteAsync(
                () => db.KeyExpireAsync(L2ProjectionKeys.MessageIndex(messageId), SlotTtl()), limit, ct);   // D-06 refresh
        }
        // retire exhaust → "do nothing" (A18 line 192): the slot stays; a future replay re-sends (dup-tolerant, A16).
    }

    // RECOV-03 tail — mutual exclusion.
    var anyInfra = temp.Any(t => t.Infra);
    if (anyInfra)
    {
        await SendKeeper(BuildReinject(d), limit, ct);   // replay owns the lifecycle; do NOT delete source
        return;
    }
    if (!SourceStep.IsSource(d.EntryId))
    {
        var del = await RetryLoop.ExecuteAsync(
            () => db.KeyDeleteAsync(L2ProjectionKeys.ExecutionData(d.EntryId)), limit, ct);
        if (!del.Succeeded) await SendKeeper(BuildDelete(d), limit, ct);
    }
    ```

    - `Guid.Empty.ToString()` is the retire value (A1 of RESEARCH; matches the slot value's `:D`/default format used in forward `entryId.ToString("D")`).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pipeline → L2 (Redis) | recovery re-reads the slot array; an L2 fault on the per-entry exist check must route to infra (preserve the slot for replay), never silently drop a recoverable entry. |
| pipeline → bus (send) | the recovery re-send of `completed` must land BEFORE the slot is retired, or a crash window could lose the result. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-51-08 | Data-integrity / Repudiation | slot retired before its `completed` result is delivered → replay never re-sends → silent result loss | mitigate | SLOT-03 send-before-retire: `SendResult` (which throws on send-exhaust) precedes the `HashSetAsync(guid.empty)`; reaching the retire == confirmed send. Asserted by the SLOT-03 ordering + send-fail-no-retire facts. |
| T-51-09 | Data-integrity | source deleted while an infra_entryId entry still needs replay → input lost before recovery completes | mitigate | RECOV-03 mutual exclusion: any infra_entryId → REINJECT and `return` BEFORE the source delete; the source delete runs only on the all-clear path. | T-51-10 | Spoofing / observability | recovery re-send reusing origin/stale exec masks the replay in telemetry | mitigate | D-03: recovery `completed` mints a FRESH `NewId.NextGuid()` exec; asserted `!= d.ExecutionId && != Guid.Empty`. Safe under A16 (orchestrator does not dedup on exec). |
