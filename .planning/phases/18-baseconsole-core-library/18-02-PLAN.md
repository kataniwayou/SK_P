       public sealed class InboundCorrelationConsumeFilter(
           ICorrelationAccessor accessor, ILogger<InboundCorrelationConsumeFilter> logger)
           : IFilter<ConsumeContext>

           public async Task Send(ConsumeContext context, IPipe<ConsumeContext> next)
           {
               var corrId = context.CorrelationId?.ToString() ?? Guid.NewGuid().ToString();

       }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inbound message → console | The correlation id arrives from an external message (MassTransit envelope `ConsumeContext.CorrelationId`) — untrusted text. |
| ambient correlation → outbound envelope | The AsyncLocal id is stamped onto outgoing envelopes; only Guid-parseable ids reach the wire. |
| appsettings → bus | RabbitMq host/credentials read from config; never logged. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-04 | Tampering / Repudiation (log injection) | InboundCorrelationConsumeFilter MEL scope | mitigate | The inbound id is placed only as a scope VALUE under the fixed `CorrelationKeys.LogScope` key — never interpolated into a format string or used as a log message template, so it cannot inject log structure. The HTTP edge already rejects CR/LF/control chars upstream; the filter treats it as opaque text. The fallback `Guid.NewGuid()` guarantees a bounded value when the envelope id is absent. |
| T-18-05 | Information Disclosure | RabbitMq credentials in MessagingServiceCollectionExtensions | mitigate | Credentials are read via `cfg.Require(...)` into the `c.Host(...)` configurator only; never written to a log/exception (the missing-key exception names the key, not the value). |
| T-18-06 | Spoofing / EoP | RabbitMQ guest/guest over non-localhost | accept | Compose-internal dev only; real creds + network policy are a Phase 19+ deployment concern (RESEARCH Security domain). No code mitigation in this library phase. |
| T-18-07 | Information Disclosure | OTel traces resurrection leaking spans | mitigate | No `.WithTracing` / `TracerProvider` / `AddSource` — asserted absent by Task 1 acceptance criteria and Plan 04's "no TracerProvider resolvable" test (Pitfall 1). |
