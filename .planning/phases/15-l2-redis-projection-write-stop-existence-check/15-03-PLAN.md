<objective>
Create the shared `RedisL2Cleanup` routine (D-06/D-07) that, for one workflowId, GETs the root key, BFS-walks the step graph in Redis (GET-and-follow, cycle-safe), collects every reachable per-step key, then batch-deletes the root + per-step keys — NEVER processor keys. The routine is ALWAYS tolerant (absent root → no-op; dangling step → skip). Also create the `OrchestrationStopException` that produces a 422 listing the full missing-workflowIds set (D-04).

Purpose: This routine has two callers (Plan 04): the Stop endpoint (runs only after the D-04 all-exist gate) and Start's pre-clean (tolerant). Building it as a shared, always-tolerant service keeps the 422 gate in `StopAsync` and out of the routine (Pitfall D). It is the GC for shrunk graphs (ORCH-START-05 delete-then-write).
Output: 1 interface, 1 impl, 1 exception, 1 integration-test fact class.

Cleanup pattern (RESEARCH Pattern 2, lines 286-318; PATTERNS.md lines 132-146):
```csharp
var db = _multiplexer.GetDatabase();
var rootJson = await db.StringGetAsync(RedisProjectionKeys.Root(prefix, wf));
if (rootJson.IsNullOrEmpty) return;                       // tolerant: absent root → no-op
var entryStepIds = JsonSerializer.Deserialize<WorkflowRootProjection>(rootJson!)!.EntryStepIds;

var visited = new List<Guid>();                           // List, not HashSet (mirror loader)
var stepKeysToDelete = new List<RedisKey>();
var currentWave = entryStepIds.Where(id => id != Guid.Empty).Distinct().ToList();
while (currentWave.Count > 0) {
    var toLoad = currentWave.Where(id => !visited.Contains(id)).Distinct().ToList();
    if (toLoad.Count == 0) break;
    var nextWave = new List<Guid>();
    foreach (var stepId in toLoad) {
        visited.Add(stepId);
        var key = RedisProjectionKeys.Step(prefix, wf, stepId);
        var stepJson = await db.StringGetAsync(key);
        if (stepJson.IsNullOrEmpty) continue;             // dangling step → skip+continue (D-06 / Pitfall 4)
        stepKeysToDelete.Add(key);
        var next = JsonSerializer.Deserialize<StepProjection>(stepJson!)!.NextStepIds;
        nextWave.AddRange(next.Where(id => !visited.Contains(id)));
    }
    currentWave = nextWave.Distinct().ToList();
}
var batch = db.CreateBatch();
var delTasks = new List<Task> { batch.KeyDeleteAsync(RedisProjectionKeys.Root(prefix, wf)) };
if (stepKeysToDelete.Count > 0) delTasks.Add(batch.KeyDeleteAsync(stepKeysToDelete.ToArray()));  // Pitfall 7
batch.Execute();
await Task.WhenAll(delTasks);
```
KeyDeleteAsync(RedisKey[]) auto-selects UNLINK on the 7.4.x server (non-blocking) — satisfies the
UNLINK-style discretion. NEVER delete {processorId} keys.
