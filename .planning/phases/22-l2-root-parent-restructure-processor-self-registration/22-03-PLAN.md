On the WebApi writer/service side: (1) make the Start writer `SADD` each workflow id into the
parent-index SET `skp:` (L2IDX-01) and REMOVE the per-processor write loop so the writer creates zero
`skp:{procId}` keys (PROC-NOCREATE-01); (2) make cleanup `SREM` the workflow from the parent index,
hoisted ABOVE the absent-root early-return so the GC is idempotent (D-10/D-12 caveat); (3) remove the
configurable `KeyPrefix` from `RedisProjectionOptions`, `BaseApi.Service/appsettings.json`, and every
read site (writer line 55, cleanup line 38, `OrchestrationService` lines 64/98/206) — switching all key
construction to the no-prefix builders from Plan 01 (L2PREFIX-01).

prefix de-config. File set is disjoint from the reader-side plan (02) — both run in Wave 2. Plan 04
(validator) edits `OrchestrationService.cs` + DI again, so it follows this plan (Wave 3).

Output: Writer SADDs the parent index + creates no processor keys; cleanup SREMs; no configurable
prefix in the writer/service code path or BaseApi.Service appsettings.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| writer → shared parent-index key | The writer SADDs untrusted-at-rest workflow-id values into the single shared `skp:` SET key |
| config → key construction | Configurable `KeyPrefix` path removed on the writer/service side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-05 | Tampering | parent-index `SADD`/`SREM` member values | mitigate | Members are strongly-typed `Guid` rendered via `ToString("D")` — no free-form string from any external caller reaches the SET. The key itself (`ParentIndex()`) is the fixed const. Worst-case stale member is reconciled by Phase 23 reload; SREM is idempotent and hoisted so GC always runs. |
| T-22-06 | Denial of service | shared `skp:` SET unbounded growth | accept | Growth is bounded by the number of active workflows the operator Starts; removal-on-Stop (SREM-in-flow) is Phase 23. Writer-side SADD is idempotent (no duplicate growth on re-Start). Low risk for the single-operator deployment. |
| T-22-07 | Tampering | writer/service key construction via config | mitigate | Removing `KeyPrefix` from options + appsettings removes the last config input to writer/service key names; the prefix is a compile-time const (Plan 01). No caller can influence the constructed key. |
| T-22-08 | Elevation of privilege | removal of per-processor write | accept | Writer no longer creates processor entries — this REDUCES write surface (PROC-NOCREATE-01). Processor entries are now owned solely by external self-registration (validated read-side in Plan 04). No new privilege path. |
</threat_model>
