<objective>
Redesign the orchestrator Start/Stop consumers to be conditionless and never-drop. Gate-open Start unconditionally hydrates+reschedules (no existence skip, no stripe); gate-open Stop unconditionally unschedules the Quartz job but KEEPS the L1 entry (drain); gate-closed Start AND Stop THROW `GateClosedException` so the scheduled-redelivery middleware reprocesses them after `MarkReady`.

Purpose: ORCH-START-RELOAD-01 (conditionless reload — supersedes Phase 23 ORCH-CONSUME-01), ORCH-STOP-DRAIN-01 (delete jobs, keep L1 for drain — supersedes Phase 23 ORCH-STOP-01), ORCH-GATE-01 (gate-closed never-drop for Start/Stop — supersedes Phase 23 D-12 gate-drop). Duplicate-suppression now lives in the WebApi (Plan 02), so the consumers need NO idempotency logic.
Output: split `WorkflowLifecycle` (add `UnscheduleOnlyAsync`); rewritten conditionless Start/Stop consumers with gate-closed throw; redelivery-before-retry added to both definitions; reconciled lifecycle tests.
</objective>
