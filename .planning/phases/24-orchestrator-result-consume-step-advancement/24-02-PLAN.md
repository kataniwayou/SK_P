**StartAsync (`OrchestrationService.cs:108-186`):**
- Keep `ExistenceCheckAsync` (404 gate) and the once-resolved `correlationId` — unchanged.
- Add a first-win existence probe per workflowId BEFORE the pre-clean: do an atomic claim on the root key. Because `RedisProjectionWriter.UpsertAsync` mints the `jobId`/builds the root JSON, the cleanest seam is: add a `db.StringSetAsync(RedisProjectionKeys.Root(workflowId), "", when: When.NotExists)` claim is INSUFFICIENT (it would write an empty/placeholder root). Instead use a pre-check `created`-gate: `var alreadyPresent = await db.KeyExistsAsync(RedisProjectionKeys.Root(workflowId));` and if `alreadyPresent`, `continue` (skip pre-clean + load + validators + Upsert for this id) AND exclude this id from the published list. Track the ids actually written into a `var started = new List<Guid>();` and `started.Add(workflowId)` only for ids that proceed.
  - RATIONALE for the pre-check (not `When.NotExists` on the real root): the real root JSON is built inside `UpsertAsync` with a fresh `jobId`; a `When.NotExists` placeholder claim followed by `UpsertAsync`'s unconditional overwrite would defeat first-win. The `KeyExistsAsync`-then-skip keeps the write path single-owner. NOTE the small TOCTOU window between the check and the write is acceptable this phase (single active WebApi replica assumed; concurrency dedup deferred per ORCH-SCALE-01). Document this with a code comment referencing WEBAPI-SUPPRESS-01 + the deferred concurrency note.
- After the loop, publish `StartOrchestration` ONLY if `started.Count > 0`, and publish `new StartOrchestration(started.ToArray())` (NOT the full input list) so a request that is entirely duplicate emits NO `StartOrchestration` (acceptance: "a second Start for an existing workflowId does not re-emit to the orchestrator").

**StopAsync (`OrchestrationService.cs:199-263`):**

Replace with delete-if-present: for each workflowId run the tolerant `_cleanup.StopCleanupAsync(workflowId, ct)` (it tolerates already-absent keys); track whether the root was present so the publish is gated. Cleanest: `var rootDeleted = await db.KeyDeleteAsync(RedisProjectionKeys.Root(workflowId));` BEFORE `StopCleanupAsync` (so the bool reflects was-present), collect into `var stopped = new List<Guid>();` adding only ids where `rootDeleted`.
