  <behavior>
    - True cycle A→B→C→A → throws Cycle with stepChain containing the cycle path (the repeated node bounds it).
    - Self-loop A→A → throws Cycle.
    - Diamond DAG A→B, A→C, B→D, C→D (acyclic, D reached via 2 paths) → does NOT throw (D-14 / Pitfall 2 — two-set algorithm required).
    - NextStepId not present in snapshot.Steps → throws MissingStep(parent, missingChild) before/instead of treating it as a node.
    - Step with null or empty NextStepIds → terminal, no throw.
    - Multiple entry steps share fullyVisited so a cleared subtree is not re-walked.
  </behavior>
  <action>
    Replace the no-op body of `CycleDetector.Validate(WorkflowGraphSnapshot snapshot)` with a two-set
    iterative DFS (D-14 — NOT the single-`visited` sketch in D-07; a single set false-positives on
    diamonds). NO recursion — explicit Stack only.

    Algorithm:
    - `var fullyVisited = new HashSet<Guid>();` (completed subtrees, shared across all entry seeds).
    - For each workflow in `snapshot.Workflows.Values`, for each `entryId in workflow.EntryStepIds ?? Enumerable.Empty<Guid>()`:
      - If `fullyVisited.Contains(entryId)` continue.
      - If `!snapshot.Steps.ContainsKey(entryId)` → `throw OrchestrationValidationException.MissingStep(Guid.Empty, entryId);` (an EntryStepId that does not resolve is a missing step; use Guid.Empty as the parent sentinel since there is no parent step — document this with a comment).
      - Run an explicit-stack DFS from entryId.
    - DFS frame: use `Stack<(Guid step, IEnumerator<Guid> children)>` plus a parallel `onStack` HashSet<Guid> AND a `path` List<Guid> to reconstruct stepChain. On push of `step`: add to `onStack` and `path`. Get children via `(snapshot.Steps[step].NextStepIds ?? new List<Guid>()).GetEnumerator()`.
    - Main loop while stack not empty: peek top frame; if `children.MoveNext()` → `var child = children.Current;`
      - If `!snapshot.Steps.ContainsKey(child)` → `throw OrchestrationValidationException.MissingStep(currentStep, child);` (D-08 missing-step gate; child referenced but absent).
      - Else if `onStack.Contains(child)` → cycle: reconstruct the chain as the slice of `path` from the first occurrence of `child` to the end, then append `child` to close the loop, and `throw OrchestrationValidationException.Cycle(cycleChain);`
      - Else if `!fullyVisited.Contains(child)` → push a new frame for `child` (add to onStack + path).
      - Else (child already fullyVisited) → skip (legal shared subgraph — the two-set discriminator; this is what prevents the diamond false-positive).
    - When `children.MoveNext()` is false → pop the frame: remove the step from `onStack`, remove it from the tail of `path`, add it to `fullyVisited`.
    - Terminal: a step whose NextStepIds is null/empty yields an empty enumerator → it is popped immediately and added to fullyVisited (passes).

    Keep CycleDetector `internal sealed class`, signature `public void Validate(WorkflowGraphSnapshot snapshot)` UNCHANGED. Add `using BaseApi.Service.Features.Orchestration;` if needed for the exception type (same namespace root — confirm). Add an XML doc summary noting: two-set (onStack + fullyVisited) iterative DFS, NO recursion (StackOverflowException is uncatchable by IExceptionHandler), reconstructs stepChain for D-03, owns BOTH the cycle (L1-VALIDATE-03) and missing-step (L1-VALIDATE-04 / D-08 defense-in-depth) gates.

    Verify there is no `Validate(` self-call and no helper method calling itself — grep must confirm no recursion.
