<objective>
Ship the two control contracts and the LOAD-BEARING scheduler change that the whole three-state model
rests on. Today every trigger is built with `TriggerBuilder.Create()` and NO `.WithIdentity(...)`, so
Quartz assigns a random GUID name and `GetTriggerState(TriggerKey(jobId))` returns `None` — the model
breaks silently. This plan stamps a deterministic `TriggerKey(jobId.ToString("D"))` on BOTH builder
sites (`ScheduleAsync` AND the per-fire self-reschedule `RescheduleAsync`) and adds the thin
`PauseAsync` / `GetTriggerStateAsync` wrappers that Plan 03's consumers call.

Purpose: Establish the Quartz-owned Running/Paused/Stopped state (D-05/D-08) and the shared message
vocabulary (D-01). Prerequisite for both Plan 03 (orchestrator consumers) and Plan 04 (Keeper publish).
Output: Two records + an extended scheduler; turns the Plan-01 scheduler/contract tests from
compile-RED to GREEN.
</objective>
