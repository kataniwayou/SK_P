### WR-01: Start-path R2 compensation can over-SREM and remove a concurrently/previously valid parent-index entry

**Files modified:** `src/BaseApi.Service/Features/Orchestration/OrchestrationService.cs`
**Commit:** 5f794bc
**Applied fix:** Rewrote the R2 comment at the Upsert catch block (formerly lines 207-212) to
correct the misleading attribution of the parent-index consistency guarantee. The comment now
states explicitly that the ACTUAL enforcing operation on the normal first-win flow is the
pre-clean `StopCleanupAsync` (step 2), which unconditionally SREMs this id from the parent index
BEFORE any write runs — so the catch-block SREM is a NO-OP in the normal flow and serves only as
a defensive belt-and-suspenders for any future/alternate path that writes without a preceding
pre-clean. Added an explicit warning not to remove the pre-clean SREM on the assumption the catch
covers it. No code changed.

### WR-02: `RedisL2Cleanup.StopCleanupAsync` is not atomic across its full sequence

**Files modified:** `src/BaseApi.Service/Features/Orchestration/Projection/RedisL2Cleanup.cs`
**Commit:** b21eef1
**Applied fix:** Took option (b) from the review — kept the current ordering and tightened the
doc comment above the delete batch (formerly lines 80-90). Added an explicit "ATOMICITY SCOPE"
paragraph stating that ONLY the final root+steps delete batch is atomic, that the parent-index
SREM at line 45 is a separate, earlier, non-batched op, and therefore the routine is NOT atomic
across its full SREM → discover-GETs → delete-batch sequence. Documented that consistency across
the SREM→delete window is caller-enforced (StopAsync catches the RedisException and SADDs the id
back via R2 compensation), not self-contained, and noted the Start-path pre-clean caller rethrows
without re-adding (acceptable because that path fails the request anyway). Did NOT take option (a)
(moving the SREM into the batch) as that would alter the locked runtime atomicity logic and is
explicitly forbidden by the line-44 idempotency comment. No code changed.
