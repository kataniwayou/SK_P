<objective>
Rework the processor receiver (`EntryStepDispatchConsumer`) into the effect-first, content-addressed shape: (1) drop-on-Ack dedup gate at the top keyed on `flag[dispatch.H]`; (2) source-step input-skip keyed on `InputDefinition == null` (the `EntryId`-empty branch is GONE); (3) per-result content-addressed write at `data[hash(blob)]`; (4) assemble a manifest `[hash(r1)...]`, write it at `data[hash(manifest)]`, compute the outbound `resultH`, pre-write `flag[resultH]=Pending` (the processor IS the sender of `ExecutionResult`, D-06), and send ONE `ExecutionResult` carrying that manifest EntryId AND `H = resultH`; (5) flip `flag[dispatch.H]` Pending->Ack via `StringSet(...,When.Exists)` AFTER the sends, then broker-ack.

Purpose: this is the producer half of exactly-once-effect — content addressing makes a retry reproduce the SAME keys (idempotent overwrite), the manifest collapses N results into one deterministic result identity, the outbound `flag[resultH]=Pending` pre-write seeds the orchestrator's drop-on-Ack gate (Plan 04 flips it Pending->Ack via `When.Exists` — without this seed that flip is a no-op and the orchestrator hop is dead), and the effect-first flag converts the only unavoidable failure (crash between effect and flip) from a LOSS into a downstream-collapsed DUPLICATE (D-06).

Output: reworked consumer + `EffectFirstDedupFacts` (req-3 processor write + req-4 processor-hop dedup + outbound Pending pre-write).
</objective>

```csharp

// SENDER pre-write (D-06): the processor is the sender of ExecutionResult, so it seeds the OUTBOUND result's
// flag[resultH]="Pending" so the orchestrator's effect-first When.Exists flip (Plan 04) has a key to flip Pending->Ack.
// Unconditional set (idempotent on re-send); Redis fault is INFRA -> propagates. Mirrors StepDispatcher's flag pre-write.
var resultH = MessageIdentity.ComputeH(dispatch.CorrelationId, dispatch.WorkflowId, dispatch.StepId, dispatch.ProcessorId, manifestEntryId);
await db.StringSetAsync(L2ProjectionKeys.Flag(resultH), "Pending", expiry: TimeSpan.FromSeconds(opts.ExecutionDataTtlSeconds));

await SendResult(BuildCompleted(dispatch, NewId.NextGuid(), manifestEntryId, resultH));   // EVEN WHEN manifestHashes is empty -> "[]" terminal (req-3)
```
Remove the old `if (built.Count == 0) return;` empty early-return — an empty result now sends the terminal manifest result so the orchestrator observes-and-acks.

   - If a result blob FAILS output-schema validation, keep today's behavior: that is a business Failed for the dispatch — send `BuildFailed(dispatch, ...)` (EntryId="", H="") and return, do NOT include it in the manifest, and do NOT pre-write a Pending flag (a Failed result short-circuits before the manifest read; see step 6 + Open Q1). (Output-validation failure is a whole-dispatch business Failed, mirroring the current per-result `BuildFailed` add; simplest faithful preservation: on first output-validation failure, SendResult(BuildFailed(...)) and return.)

5. **Effect-first flag flip (D-06), then ack:** BuildCompleted/the send (including the outbound `flag[resultH]=Pending` pre-write from step 4) happen FIRST; immediately before the normal return add:
```csharp
// Effect-first CAS (D-06/D-07): flip the INBOUND dispatch's flag Pending->Ack only after the effect (write+send) is produced.
// When.Exists = SET XX: a false return means Pending was lost (crash residual) — NOT an error, do NOT throw (Pitfall 3).
await db.StringSetAsync(L2ProjectionKeys.Flag(dispatch.H), "Ack", when: When.Exists);
```
(The OUTBOUND `flag[resultH]=Pending` seed is part of "producing the effect" — it accompanies the send in step 4. The receiver-side `Pending->Ack` flip stays effect-first: the inbound `flag[dispatch.H]` is flipped only here, after the send. Do NOT flip the outbound `resultH` to Ack — the orchestrator owns that flip on its hop, Plan 04 Task 2.)

6. **Builders:** `BuildCompleted(d, executionId, manifestEntryId, resultH)` sets `EntryId = manifestEntryId` AND `H = resultH` (the caller passes the already-computed `resultH` so the pre-write and the message carry the IDENTICAL H — one ComputeH call, no drift). `BuildFailed`/`BuildCancelled` set `EntryId = ""` and `H = ""` (a Failed/Cancelled result fans to no Completed-gated successor; the orchestrator short-circuits on outcome before reading the manifest — Open Q1 decision; an empty H also means no outbound Pending pre-write is needed for those, consistent with step 4).

Preserve the INFRA-throw vs BUSINESS-ack split everywhere: flag get/set (including the outbound Pending pre-write) + data write + Send are INFRA (no catch); ProcessAsync exceptions + schema failures stay caught-and-acked.
