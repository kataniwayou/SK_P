<objective>
Rework the orchestrator dispatch side for exactly-once-effect: (1) entry-step `EntryId = hash(correlationId, stepId)` stamped in `WorkflowFireJob` (req-2); (2) inbound `ExecutionResult` dedup on `flag[m.H]` effect-first in `ResultConsumer`; (3) manifest unbundle + N×M fan-out (read `data[m.EntryId]` -> `string[]` -> for each item × each `SelectNext` successor, dispatch with the item EntryId + a deterministic child H); (4) `StepDispatcher` computes and stamps the deterministic `H` per dispatch and pre-writes `flag[H_child] = Pending` (the sender-pre-write half of D-06, symmetric on both hops).

Purpose: this is the consumer/fan-out half of exactly-once-effect. The deterministic child `H` (executionId excluded) means an orchestrator redelivery reproduces the same `H` per (item, successor) and is deduped — no extra dispatch (req-6). Merge correctness falls out for free: different input `EntryId` -> different `H` (per-edge, no false dedup, no override); identical input -> same `H` -> collapse (req-5).

Output: reworked `WorkflowFireJob`/`ResultConsumer`/`StepDispatcher` + `ManifestFanoutFacts` + `MergeCollapseFacts` + the deferred FireDispatch req-2 assertions.
</objective>
