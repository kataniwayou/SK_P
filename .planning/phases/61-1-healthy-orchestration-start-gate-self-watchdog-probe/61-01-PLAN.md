| external self-registrant -> L2 per-instance value | A processor replica writes its own `skp:proc:{procId}:{instanceId}` value; the gate reads data it does not own (untrusted shape). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-61-01 | Denial of Service | ProcessorLivenessValidator deserialize of external value | mitigate | Malformed JSON / null Summary is caught and counted as `malformed` (fails that replica) -> 422, NEVER an escaping JsonException/NRE that the caller's RedisException catch misses and turns into a 500. Verified by MalformedProcessorRegistration_Returns422 (Task 3). |
| T-61-02 | Information Disclosure | 422 `offending.reason` aggregate string | mitigate | The reason carries per-state COUNTS + the processor Guid only — never instanceIds, connection strings, or stack traces (T-14-02 / V7). Format string `"no healthy replica (N checked: A absent, U unhealthy, S stale, M malformed)"` is count-only by construction. |
| T-61-03 | Tampering / DoS | genuine RedisException on SMEMBERS/GET | accept (existing contract) | A real transport fault propagates untouched -> 500 via the caller's `OrchestrationService.cs:197` catch (NOT the validator). The validator adds no RedisException catch; only deterministic data states map to 422. This 422-vs-500 split is the load-bearing invariant. |
