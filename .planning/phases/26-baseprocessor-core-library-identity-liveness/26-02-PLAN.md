<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| processor -> WebApi (over RabbitMQ bus) | the identity/schema request crosses the bus; the response is an untrusted-but-typed message |
| host startup -> dependency availability | the WebApi responder / DB row may be absent at boot (boot-before-register) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-04 | Denial of Service | unbounded retry loop (resource exhaustion) | mitigate | Bounded exponential backoff capped at `BackoffCapSeconds` (~30s) between attempts via `Math.Min(delay*2, cap)`; exactly one in-flight request at a time per loop (HydrationBackgroundService:81 pattern). One short per-request timeout (`RequestTimeoutSeconds` ~5-10s) so a slow/absent responder fails fast and re-loops rather than holding resources. |
| T-26-05 | Denial of Service | host crash on dependency fault (WebApi/DB absent) | mitigate | NotFound + RequestTimeoutException are caught and looped (boot-before-register tolerated, D-04) — the BackgroundService never rethrows on a resolution failure, so the host stays up. Only OperationCanceledu on shutdown returns. |
| T-26-06 | Spoofing | a non-Healthy replica appearing live | mitigate | The gate (`MarkReady`) and `MarkHealthy()` fire ONLY after identity + all required definitions resolve; until then `/startup` and `/ready` stay red and the Plan 03 heartbeat (which gates on IsHealthy) writes nothing — a non-Healthy replica is `absent` to the orchestrator. |
| T-26-07 | Tampering | untrusted response message shape | accept | Responses are MassTransit-typed dual-response contracts (no free-form parsing); a malformed message fails deserialization at the transport, surfacing as a timeout that the loop retries. Schema-definition CONTENT validation is Phase 27, not this phase. |
</threat_model>
