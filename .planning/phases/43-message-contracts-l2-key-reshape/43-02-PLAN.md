## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-43-03 | Tampering (sentinel confusion) | Guid.Empty source sentinel on EntryStepDispatch / Step* records | mitigate | A single shared predicate (SourceStep.IsSource, D-07) is the ONLY recognizer; ExecutionLogScope and (Plan 03) the processor read both route through it. Plan 01's StepResultContractTests pins that the sentinel serializes to the literal all-zero GUID, so a reader cannot mistake a present zero-GUID for an omitted field. |
| T-43-04 | Information disclosure | ExecutionLogScope.BuildState writing entryId into log scope | mitigate | The IsSource guard suppresses the all-zero GUID for source steps (Pitfall 2), so no spurious zero-GUID identifier leaks into structured logs. EntryId is an internal correlation GUID, not PII or a secret. |
| T-43-05 | Denial of service (STJ deserialization) | New sealed records with positional + init members | accept | Records use default STJ with no custom converters or recursion; no untrusted-size collections. The contract surface matches the existing v3.x records (same deserialization risk profile, no new external boundary). ASVS L1: no high-severity finding. |
</threat_model>
