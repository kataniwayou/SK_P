| dispatch consumer <- bus (untrusted message body) | EntryStepDispatchConsumer reads `dispatch.EntryId` (now a Guid) and resolves L2 data from it. The Guid.Empty sentinel must skip the read. |
| result consumer <- bus | ResultConsumer consumes StepCompleted and advances L1 state. No Redis read on this path post-change. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-43-06 | Tampering (sentinel bypass) | EntryStepDispatchConsumer input read on dispatch.EntryId | mitigate | The read-skip branch routes through SourceStep.IsSource (the single D-07 predicate), not an ad-hoc check — a Guid.Empty dispatch deterministically skips the L2 read rather than constructing a malformed key. |
| T-43-07 | Elevation / logic flaw (lost dedup) | Removal of flag[H]/CAS at both hops | accept | Loss of dedup is an INTENTIONAL milestone posture (RESIL-03: at-least-once, duplicates tolerated; "Out of Scope: any dedup/idempotency key"). Duplicate effects are tolerated downstream by design; no security boundary depends on exactly-once here. |
| T-43-08 | Information disclosure | ResultConsumer dropping the IConnectionMultiplexer dependency | accept | Removing the Redis dependency reduces the consumer's attack surface (no L2 read on the result path). No new exposure introduced. |
