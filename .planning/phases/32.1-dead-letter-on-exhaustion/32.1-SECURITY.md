# Phase 32.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

This phase is a NET-DELETION revert of the Phase-32 cancelled circuit-breaker. It REMOVES
attack surface rather than adding it. Each threat below is either a surface ELIMINATED by the
revert (`accept (now eliminated)`) or a regression we must NOT introduce while deleting
(`mitigate`). All mitigations are verified by deterministic structural greps against the
committed code.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| broker → processor consume | `EntryStepDispatch` crosses into `EntryStepDispatchConsumer`; on infra exhaustion it now dead-letters to `{processorId}_error` via `UseMessageRetry(Immediate(Limit))` (was: breaker trip + no-TTL marker write) | dispatch message (workflow/step ids) |
| broker → orchestrator consume | `ExecutionResult` / auto-published `Fault<EntryStepDispatch>` cross into the orchestrator; the Fault binding is REMOVED — the Fault is discarded at the exchange | result message / fault envelope |
| process → Redis L2 | the cancellation-marker WRITE path (no-TTL `skp:cancelled:*`) is REMOVED — net reduction of write surface | (write path eliminated) |
| operator → live stack | `scripts/phase-32.1-close.ps1` snapshots the live keyspace/db/queues — read-only SHA intent, no destructive writes (no `FLUSHDB`) | snapshot hashes only |
