<objective>
Extend the two NSubstitute test kits so the A19 multi-key (array) `KeyDeleteAsync(RedisKey[], CommandFlags)` overload and the best-effort `KeyPersistAsync(RedisKey, CommandFlags)` overload are mocked everywhere the production change will call them. Purpose: deliver the mock surface the production array DEL + persist escalation rely on, with NO production behavior change yet — the kit edits are inert until Plan 03 calls the array overload.
Output: `DispatchTestKit.cs` (array + persist stubs on tail muxes, array When/Do on fault muxes, new `ReadOkDeleteAndPersistFaultL2` sibling) and `RecoveryTestKit.cs` (array stub on `Db()`).

Verified Redis API signatures (StackExchange.Redis 2.13.1, from the pinned assembly XML):
```
KeyDeleteAsync(RedisKey   key,  CommandFlags flags = None)  -> Task<bool>   (scalar — current)
KeyDeleteAsync(RedisKey[] keys, CommandFlags flags = None)  -> Task<long>   (array — A19 target)
KeyPersistAsync(RedisKey  key,  CommandFlags flags = None)  -> Task<bool>   (A19 escalate)
```
NSubstitute treats the array overload as a DISTINCT method from the scalar one. `Arg.Any<RedisKey[]>()` matches the array overload; `Arg.Any<RedisKey>()` matches only the scalar. A recent NSubstitute returns a non-null completed `Task<long>` (wrapping `0L`) for an unstubbed array member — so a forgotten array fault stub yields a FALSE GREEN (the exhaust branch never runs).

Existing scalar stubs/When-Do (the analogs to mirror), VERIFIED against current source:
- Success-mux scalar stub: `db.KeyDeleteAsync(Arg.Any<RedisKey>(), Arg.Any<CommandFlags>()).Returns(true);`
- Fault-mux scalar When/Do (`ForwardDeleteFaultL2`, `ReadOkDeleteFaultL2`):
