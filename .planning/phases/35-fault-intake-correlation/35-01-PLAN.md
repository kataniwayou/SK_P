<objective>
Refactor the execution-scope dictionary builder out of `InboundExecutionScopeConsumeFilter` into a shared, pure-POCO `ExecutionLogScope.BuildState(IExecutionCorrelated)` helper in `Messaging.Contracts` (D-07). This is the single-source-of-truth foundation that the Wave-2 Keeper fault consumers call to open the execution log-scope manually (the bus-wide filter does NOT fire on `Fault<T>`).

Purpose: KMET-04 requires Keeper logs to carry the same execution-scope ids as every other console. Centralizing the scope-key set + skip rules prevents per-console drift. Because this touches a base library used by ALL consoles, the refactor MUST be byte-identical in observable behavior and is regression-guarded.

Output: `BuildState` static helper + the filter delegating to it; all existing scope tests still green.
</objective>
