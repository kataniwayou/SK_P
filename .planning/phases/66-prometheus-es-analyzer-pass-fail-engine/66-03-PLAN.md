REAL telemetry, feeds the pure `PassFailEngine` (Plan 01), writes the JSON report, and asserts the
verdict. This implements OBS-04 (per-test report + automated PASS/FAIL derived solely from Prom + ES)
and exercises OBS-01/02/03 end-to-end against the live stack.

It begins with the Wave-0 probes the research flagged: a `_mapping` probe to confirm the window
timestamp field (A2) + `Sum` attribute type (A1), baked into an `EsIndexNames` const + the query
template; and confirmation of the Prom counter windowing strategy (A3) — counters are lifetime
cumulative, so reconciliation uses a windowed DELTA (snapshot-before/after or `increase(metric[window])`),
never raw totals.

The fixture: (post-drain) `SearchAllHits` over the window for the `Step_*` family → group into
`RunTrace`s by `attributes.CorrelationId` → read the Prom counter set as windowed deltas → derive the
trigger denominator from `orchestrator_dispatch_sent_total` + cadence (NO per-fire correlationId log
exists — item #1) → `engine.Analyze(...)` → WRITE the JSON report → THEN `Assert.True(verdict == Pass)`.

Depends on Plan 01 (engine + models) and Plan 02 (SearchAllHits + field consts) — Wave 2.

`dotnet test --filter`, reading the exit code + the JSON report. Write-then-assert means the report
exists even on a red run.
