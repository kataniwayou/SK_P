Logic (encode exactly, per D-03..D-08 + 66-RESEARCH Counter Reality):
1. COMPLETE (OBS-01): `complete = runs.Where(r => r.DistinctLabels.SetEquals(AllLabels)).ToList();`
   A run is COMPLETE iff its distinct StepLabel set equals the full 9-label set — which necessarily
   includes BOTH sinks Step_F1 + Step_F2.
2. MISSING (OBS-02): `missing = triggerCount - complete.Count;`. The denominator triggerCount comes
   from Prom orchestrator_dispatch_sent_total + cadence (passed in by the fixture) — NOT from a
   per-fire correlationId ES log (it does not exist; item #1). The SPECIFIC missing correlationId
   is NOT recoverable from telemetry — record this as a caveat in MissingDetail, do NOT attempt to
   name it.
3. DUPLICATE (OBS-02, fail-closed): `duplicates = runs.Where(r => r.HasAnyDuplicateLabel).ToList();`.
   `dupFail = duplicates.Count > 0;` — ANY duplicate (correlationId, StepLabel) is a FAIL. There is
   NO live dedupe counter to corroborate a redelivery (orchestrator_result_deduped /
   processor_dispatch_deduped are DORMANT — null/absent in the snapshot), so an un-corroboratable
   duplicate is the only kind → fail-closed (D-06 contingency + D-07). Still surface the duplicate
   loudly in the report.
4. RECONCILE (OBS-03, D-08) on the LIVE counter set only (dormant dedupe deltas feed NO arithmetic;
   report them as Absent):
   ```csharp
   bool reconciled =
       prom.DispatchSentDelta == triggerCount
       && prom.ResultSentCompletedDelta >= complete.Count * LabelsPerRun
       && UnaccountedDelta(prom) == 0;        // dispatched-but-never-completed not explained
   var recon = reconciled ? ReconciliationOutcome.Reconciled : ReconciliationOutcome.Unreconciled;
   ```
   Define `UnaccountedDelta` = `dispatch_sent − result_consumed` (or the dispatched-vs-completed gap)
   that is NOT mirrored by reported redelivery; an UNRECONCILED imbalance is itself a FAIL (D-08
   fail-closed). Any NonCompletedOutcomes entry (failed/cancelled/processing > 0) also forces
   Unreconciled.
5. VERDICT: `pass = missing == 0 && !dupFail && recon == Reconciled;`
   `verdict = pass ? Verdict.Pass : Verdict.Fail;`
6. Build and RETURN the AnalyzerReport (scenarioId, verdict, triggerCount, complete.Count, missing,
   MissingDetail with the unrecoverable-identity caveat, duplicates, recon, prom, all traces, and a
   HumanSummary string). The engine returns the report — it performs NO file IO (the fixture writes).
