## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-05 | Tampering | OrchestratorForwardWrite Lua script | mitigate | The script is a compile-time `const` with parameterized KEYS/ARGV — NO orchestrator/user data is concatenated into the script text (mirrors the processor T-69-04 precedent). The slot tuple flows as ARGV[2], never spliced into the script body. |
| T-71-06 | Denial of Service (own state) | Malformed/retired slot JSON crashing RECOVERY | mitigate | RECOVERY parses tolerantly and SKIPS retired (guid.empty) / unparsable slots (mirror the Guid.TryParse skip at ProcessorPipeline.cs:173) rather than throwing — a bad slot cannot abort the recovery pass. |
| T-71-07 | Availability (resource leak) | Copied data key with no TTL → immortal key | mitigate | The GET+SET copy leg carries the data TTL inline (`SET KEYS[2] v PX ARGV[4]`) inside the single atomic script (D-03 / Pitfall 3); no COPY-without-PEXPIRE path exists. |
| T-71-08 | Elevation/Repudiation | Over-deletion of keys from the pipeline | mitigate | Only the gated cleanup tail (DeleteTerminalAsync analog) calls KeyDeleteAsync; FORWARD/RECOVERY escalation legs never delete. The negative-guard fact for the consumers is in Plan 04 (D-09); the pipeline's deleter is grep-confirmed to be the single cleanup-tail method (acceptance criteria, Task 1). |
