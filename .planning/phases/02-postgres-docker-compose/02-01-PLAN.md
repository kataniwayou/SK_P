<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Compose interpolation -> container env | `${POSTGRES_*}` values from committed `.env` flow into the Postgres container environment at `compose up` time. These are dev-default values (not real secrets) per PROJECT.md Out of Scope; the password slot is honest about being "auth/secrets deferred to v2" rather than pretending to be safely configured. |
| Host port 5433 -> container port 5432 | The Postgres container listens on `localhost:5433` from the dev host's perspective. Only loopback by default with the `5433:5432` mapping form (Docker binds to all interfaces by default — see mitigation below). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `.env` committed to git with `POSTGRES_PASSWORD=postgres` | accept | Per PROJECT.md "Out of Scope: Authentication / authorization" + Pitfall 39, v1 explicitly ships dev-default credentials in committed config. WR-01 from Phase 1 review is acknowledged as a v2 hardening item; not in Phase 2 scope. Mitigation deferred to v2 secret-manager integration. |
| T-02-02 | Spoofing / Tampering | Host port 5433 exposed by Docker on all interfaces (default `5433:5432` binds to `0.0.0.0`) | accept | Dev-local only; documented in README under "Local Postgres (Docker)" as a dev-only port. Production deployment (Phase 8+) lands its own compose/k8s topology with proper network isolation; this compose file is explicitly the dev-on-host loop. No production data lives here. |
| T-02-03 | Denial of Service | Postgres container restarts indefinitely under `restart: unless-stopped` | accept | Default Docker policy; bounded by the dev box's resources. Deliberate per D-09 — the developer wants Postgres back up after a reboot without `docker compose up` again. No remediation needed. |
| T-02-04 | Tampering | Docker socket exposure | n/a | This compose file does NOT mount `/var/run/docker.sock` into any container. No threat surface introduced. |
| T-02-05 | Information Disclosure | `.env.local` per-developer overrides accidentally committed | mitigate | `.gitignore` append adds both `.env.local` AND `*.env.local` (defensive — handles `staging.env.local`, `dev.env.local`, etc.). Verified by `grep -E '^\.env\.local$' .gitignore` and `grep -E '^\*\.env\.local$' .gitignore` in this plan's acceptance criteria. |
| T-02-06 | Repudiation / Elevation | n/a | n/a | No auth boundary, no user identity, no audit trail in Phase 2 — those land in later phases. The Postgres `superuser=postgres` is intentional per D-04; dedicated app user is a v2 hardening item. |

**Severity:** LOW across the board for a dev-local compose file. The threat surface is documented to satisfy the security-enforcement gate; no `high`-severity threats are introduced by this phase.
