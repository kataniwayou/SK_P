Join the WebApi to the MassTransit bus as a publish-only participant (Stream B, part 1) and publish Start/Stop from `OrchestrationService` with a body-carried, freshly-minted CorrelationId.

Covers: MSG-WEBAPI-01 (publish-only join referencing Messaging.Contracts, never BaseConsole.Core), MSG-WEBAPI-02 (Start publishes StartOrchestration{ids}, Stop publishes StopOrchestration{ids}), MSG-WEBAPI-03 (broker hard-dep for Start/Stop only — 5xx + RFC 7807 on broker-down, CRUD unaffected), MSG-WEBAPI-04 (bus health Degraded-not-Unhealthy so CRUD /health/ready stays 200).

Purpose: the publisher side of the fan-out — the WebApi mints the bus-stage CorrelationId at the publish boundary (per-stage handoff, D-01/D-02) and broadcasts the control message. Output: `AddBaseApiMessaging` in BaseApi.Core, the publish call sites in OrchestrationService, the WebApi RabbitMq appsettings section, and the in-memory publish harness test.

This plan does NOT touch compose.yaml or any Dockerfile (Plan 04). It does NOT set the MassTransit envelope CorrelationId (body-only — D-02). It does NOT add a correlation filter or AsyncLocal accessor to the WebApi (publish-only — D-02). The broker-down real-CRUD-health test is Phase 20 (TEST-RMQ-03) — this plan asserts the option is set in-process.

AddBaseApiMessaging (NEW, public static — publish-only):
```csharp
public static IServiceCollection AddBaseApiMessaging(this IServiceCollection services, IConfiguration cfg)
{
    var host = cfg.Require("RabbitMq:Host");
    var user = cfg.Require("RabbitMq:Username");
    var pass = cfg.Require("RabbitMq:Password");
    services.AddMassTransit(bus =>
    {
        bus.ConfigureHealthCheckOptions(o =>
        {
            o.MinimalFailureStatus = HealthStatus.Degraded;   // never escalate to Unhealthy (D-05)
            // DO NOT touch o.Tags — overriding REPLACES defaults ["ready","masstransit"] (Pitfall 7)
        });
        bus.UsingRabbitMq((context, busCfg) =>
        {
            busCfg.Host(host, h => { h.Username(user); h.Password(pass); });
            // NO ConfigureEndpoints — publish-only
        });
    });
    return services;
}
```
