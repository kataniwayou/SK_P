Three behaviors to prove (CONTEXT testing_guidance):
1. On the Gate-A-clash path, advancing the FakeTimeProvider by N intervals causes the per-instance
   key to be re-SET each interval with a fresh timestamp + reset TTL and status=Unhealthy (i.e. NOT
   a single write-then-stop), and L1 `Current.Timestamp` advances each interval.
2. The watchdog verdict math is unchanged — a refreshed-but-unhealthy replica (interval=10, fresh
   timestamp) reads as "live"/fresh, NOT "loop stale" (reuse/extend the watchdog verdict assertion).
3. Shutdown (`stoppingToken` cancel via `StopAsync`) exits the refresh loop cleanly.

Purpose: lock the fix against regression and prove the decoupling (timestamp-fresh, status-Unhealthy)
WITHOUT a live container lifecycle. The watchdog/heartbeat/validator stay unchanged — Behavior 2 is a
no-change assertion (a green watchdog verdict on a fresh interval=10 Unhealthy entry).

Output: a new `tests/BaseApi.Tests/Processor/ClashRefreshFacts.cs` fact class following the existing
`StartupUnhealthyWriteFacts` harness (RedisFixture + in-memory MassTransit + FakeTimeProvider + the
`IdentityResolutionFacts` stubs that drive the Gate-A clash via `GateAStubConfig`).
