## Self-watchdog (`/health/live`) semantics

| Option | Description | Selected |
|--------|-------------|----------|
| Staleness-only, no verdict change | Liveness probe answers "is the loop alive / restart me?" — refreshed-but-unhealthy reports Healthy "live", with configSchema=Fail in the body via the PROBE-02 summary. Expected: LivenessWatchdogHealthCheck needs NO change. | ✓ |
| Flip verdict to Unhealthy | Make /health/live report Unhealthy when the entry status is Unhealthy. Conflates "unhealthy" with "should restart" — the false-restart bug. | |

**User's choice:** Staleness-only, no verdict change.
**Notes:** Flipping the verdict IS the false-restart bug the gap doc warns against (Impact #2). The watchdog
already serializes the per-schema summary into the response body, so the real (unhealthy) status is observable
without changing the liveness verdict.
