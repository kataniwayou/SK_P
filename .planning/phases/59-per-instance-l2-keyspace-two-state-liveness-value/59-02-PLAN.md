| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-59-04 | Tampering / Injection | `instanceId` interpolated into the L2 key in `PerInstance` | accept | The `instanceId` comes from process-controlled sources (POD_NAME/HOSTNAME env, MachineName, or a GUID) within the same trust domain as the orchestration plane — not from an untrusted client request. Phase 59 introduces no new external input boundary. |

| T-59-06 | Information Disclosure | POD_NAME/HOSTNAME in the key | accept | Pod/host names are non-sensitive operational identifiers already exposed via OTel `service.instance.id`; no PII or secret. Internal infra store. |
