        });
    });
    return services;
}

| responder → existing read service | The consumer calls the EXISTING `ProcessorService`/`SchemaService` read paths (unchanged, parameterized EF queries). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-02-01 | Information disclosure | `ProcessorIdentityFound` returns `{ Id, InputSchemaId?, OutputSchemaId?, ConfigSchemaId? }` over the bus; `SchemaDefinitionFound` returns the schema `Definition` | accept | Responses carry exactly what the Phase 26 processor needs to resolve identity/schemas and no more — a direct field projection of `ProcessorReadDto` / `SchemaReadDto.Definition` (no secrets, no PII; schema definitions are non-sensitive contract JSON). The bus is an internal trust boundary (RabbitMQ inside the compose network), not an external surface. |
| T-25-02-02 | Information disclosure (enumeration) | not-found response leaking existence/enumeration info | accept | `ProcessorIdentityNotFound`/`SchemaDefinitionNotFound` echo only the requested key (SourceHash/SchemaId) the caller already holds — they reveal nothing beyond "no row for the key you sent." This is the intended D-04 dual-response signal for the Phase 26 retry loop; no broader enumeration than a single-key probe. |
| T-25-02-03 | Denial of service | unbounded request handling on the two responder endpoints | accept | Each request is a single indexed read (`SourceHash` lookup / PK `GetByIdAsync`) over EF Core; no fan-out, no unbounded allocation. The bus is internal (no external client can flood it). Responder-side retry/throttle is deliberately out of scope (client retry is Phase 26); no new amplification path. |
| T-25-02-04 | Tampering / EoP (integrity) | the dependency firewall (Core → Service/Console) and the Degraded health cap are integrity invariants the hook addition could breach | mitigate | `BaseApiCoreFirewallTests` (Task 1) asserts no `BaseApi.Service`/`BaseConsole.Core` reference; the `MinimalFailureStatus = Degraded` block is kept byte-identical and guarded by `Health_Ready_Returns_200_When_Broker_Dead` (Task 3). Hooks are typed in MassTransit interfaces only — Core never names a concrete consumer type. |
| T-25-02-05 | Spoofing/Repudiation | bus-message origin authenticity | accept | Out of scope this phase — the existing RabbitMQ transport trust model is unchanged; no new auth surface. The CRUD auth surface is untouched (no HTTP endpoint added). |

No threat reaches HIGH. NO new external HTTP surface is added (responders are bus-only); the CRUD auth surface is untouched. The firewall + Degraded-cap integrity guards are actively tested. **Phase not blocked.**
