<objective>
Close GAP-49-2 (severity: high): an orchestrator Quartz scheduler stuck PAUSED for newly-scheduled workflows after any global pause-all/resume-all cycle. This inverts the v4 recovery promise — a transient, recoverable L2 outage becomes a PERMANENT scheduling freeze until the orchestrator restarts. It is the sole open defect blocking the live N-consecutive-GREEN close gate (the four original phase plans 49-01…49-04 are DONE; GAP-49-1 was already resolved by commit 5666fb7).

Root cause (LOCKED, D-08): `PauseAllConsumer` calls scheduler-wide Quartz `PauseAll()`, which adds every trigger group to Quartz's `pausedTriggerGroups` set. Triggers ADDED afterward to a paused group are born `PAUSED`. `ResumeAllConsumer` resumes PER-JOB (delete-stale + fresh-from-now reschedule via `WorkflowLifecycle.ResumeAsync`) and never clears `pausedTriggerGroups`, so the resume's own fresh triggers AND brand-new workflows stay paused until restart.

Fix (LOCKED, D-08 — Option A, chosen over per-job-symmetric-pause): leave `PauseAllConsumer` / `PauseAll()` UNCHANGED (atomic, instant, outage-wide). In `ResumeAllConsumer.Consume`, AFTER the existing per-job `ResumeAsync` loop completes, call ONE group-level resume — `scheduler.ResumeTriggers(GroupMatcher<TriggerKey>.AnyGroup(), ct)` — purely to clear `pausedTriggerGroups`. The scheduler owns the Quartz call via a new `ResumeAllGroupsAsync(ct)` wrapper (mirroring `PauseAllAsync`), keeping the consumer thin.

Ordering is LOAD-BEARING: the group-level resume MUST run AFTER the per-job loop. By then every paused workflow trigger was already replaced with a fresh-from-now `Normal` trigger, so NO stale paused trigger remains to misfire-herd — the group-flag clear has no herd. This ordering is what neutralizes the herd the original `Native_ResumeAll_Is_Never_Called` lock guarded against; the binding guarantee shifts from "no group-level resume call ever" to "no immediate-refire herd."

Purpose: Unblock the operator-gated live close gate (TEST-01/02/03 stay unticked until the operator's GREEN run, D-03; this plan removes the defect that sinks that run).
Output: 2 production-file edits (scheduler wrapper + consumer call) + 2 test-file edits (replaced no-burst contract + new GAP-49-2 regression).

Net-zero / scope discipline: `PauseAllConsumer.cs` and `PauseAllConsumerTests.cs` stay UNCHANGED. Do NOT touch the SC1/SC2/SC3 E2E files or `scripts/phase-49-close.ps1` — they are already verified (49-VERIFICATION.md 4/4 truths). This plan is the orchestrator fix + its unit/regression tests ONLY.

```csharp
public async Task ResumeAsync(Guid workflowId, CancellationToken ct)
{
    if (!store.TryGet(workflowId, out var wf)) return;            // business no-op
    var state = await scheduler.GetTriggerStateAsync(wf.JobId, ct);
    if (state != TriggerState.Paused) { /* log + return — ignore non-Paused */ return; }
    await scheduler.UnscheduleAsync(wf.JobId, ct);                // DeleteJob (removes stale paused job)
    await scheduler.ScheduleAsync(workflowId, wf.JobId, wf.Cron, ct); // fresh from-now trigger -> Normal
}
```
